# Roadmap

Living record of where Soccer Manager is going. This tracks **high-level goals** —
not individual tickets. When something here grows into real work, it may spawn a
GitHub issue or a `backlog/` task; this stays at the altitude of *"what and why."*

Vision lives elsewhere and does not belong here:

- **[`PRODUCT.md`](../../PRODUCT.md)** — who it's for, product purpose, brand, principles.
- **[`docs/design-brief.md`](../design-brief.md)** — the visual/design direction.
- **[`README.md`](../../README.md)** — repo layout and commands.

## Files

- **[`product.md`](./product.md)** — features for the game (what a manager can *do*).
- **[`project.md`](./project.md)** — infra, deploy, repo structure, tooling, quality.

## Status legend

| | Status | Meaning |
|---|---|---|
| 🟢 | **Done** | Shipped and in use. |
| 🔵 | **In progress** | Actively being built. |
| 🟡 | **Next** | Agreed and queued — the near-term focus. |
| ⚪ | **Backlog** | Wanted, not yet scheduled. |
| 💭 | **Idea** | Discussed / under consideration; not committed. |

## Conventions

- Keep entries **high-level** — one line of *what*, one of *why* if not obvious.
- Order within a section roughly by priority (top = sooner).
- When a goal ships, mark it 🟢 rather than deleting it — the "Done" trail is useful.
- Prefer editing an existing line over adding a near-duplicate.
